# Цифровий пазл (Тестове завдання)

Програма для пошуку найдовшого ланцюжка з цифрових фрагментів. Фрагменти склеюються між собою, якщо останні дві цифри одного збігаються з першими двома цифрами наступного. Кожен шматок використовується лише один раз.

## Як це працює всередині

1. **Швидкий пошук:** Програма збирає всі фрагменти у граф (масив списків), де індексами є числа від 00 до 99. Це дозволяє миттєво знаходити потрібне продовження для пазла.
2. **Розумний старт:** Алгоритм шукає "голову" ланцюжка — фрагмент, який сам нікого не продовжує. Це економить купу часу процесора, бо ми не починаємо перебір з середини.
3. **Обхід у глибину (DFS):** Програма йде по графу до самого кінця, фіксує довжину і вертається назад для перевірки інших варіантів.

## Запуск

Програма написана на чистому Node.js і не потребує встановлення сторонніх бібліотек.

Переконайся, що файл `source.txt` лежить в корені поруч з core, parser!
## Вибір режиму виконання
Проєкт підтримує два режими виконання:
- **Singlethreaded:** `cd singlethreaded && node main.js` (підходить для швидких запусків).
- **Multithreaded:** `cd multithreaded && node main.js` (використовує паралельні потоки для прискорення на великих файлах).
